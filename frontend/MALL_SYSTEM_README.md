# 商城系统使用说明

### 1. 商城页面 (`/mall`)

- ✅ 分类筛选
- ✅ 排序功能（综合、销量、价格）
- ✅ 库存筛选
- ✅ 商品详情弹窗
- ✅ 加入购物车
- ✅ 加入心愿单
- ✅ 分页功能
- ✅ 响应式设计

### 2. 购物车页面 (`/cart`)
- ✅ 购物车商品列表
- ✅ 全选/单选功能
- ✅ 数量调整
- ✅ 删除商品
- ✅ 移入心愿单
- ✅ 订单摘要（总价、运费计算）
- ✅ 结算功能
- ✅ 推荐商品
- ✅ 空购物车状态
- ✅ 响应式设计

### 3. 订单页面 (`/orders`)
- ✅ 订单列表展示
- ✅ 订单状态筛选（全部、待支付、待发货、待收货、已完成、已取消）
- ✅ 订单详情展示
- ✅ 支付功能
- ✅ 查看物流
- ✅ 确认收货
- ✅ 取消订单
- ✅ 评价功能（待开发）
- ✅ 响应式设计

## 数据存储

系统使用 `localStorage` 存储数据：

### 存储键值
- `cart` - 购物车数据
- `orders` - 订单数据
- `wishlist` - 心愿单数据（商品ID数组）

### 数据结构

**购物车项 (CartItem)**
```typescript
{
  id: number
  name: string
  price: number
  quantity: number
  icon: string
  color: string
  category: string
  designer?: string
  selected: boolean
}
```

**订单 (Order)**
```typescript
{
  id: string
  orderNumber: string
  createTime: string
  status: 'pending' | 'paid' | 'shipped' | 'completed' | 'cancelled'
  items: OrderItem[]
  totalAmount: number
}
```

## 业务流程

### 购物流程
1. 用户在商城浏览商品
2. 点击"加入购物车"或打开详情后加入
3. 进入购物车页面
4. 选择商品、调整数量
5. 点击"去结算"
6. 自动创建订单并跳转到订单页面

### 订单流程
1. 订单创建（状态：待支付）
2. 用户支付（状态：待发货）
3. 商家发货（状态：待收货）
4. 用户确认收货（状态：已完成）
5. 用户评价（可选）

## 特色功能

### 1. 运费计算
- 订单满 99 元免运费
- 不满 99 元收取 10 元运费

### 2. 商品推荐
- 购物车页面显示推荐商品
- 点击推荐商品可直接加入购物车

### 3. 心愿单
- 支持将商品加入心愿单
- 购物车商品可移入心愿单

### 4. 响应式设计
- 完美适配桌面端、平板、手机
- 移动端优化的交互体验

## 注意事项

- 所有数据存储在浏览器 localStorage 中
- 清除浏览器数据会导致购物车和订单数据丢失
- 当前为前端演示版本，实际生产环境需要对接后端 API
- 支付功能为模拟实现，实际需要对接支付网关
