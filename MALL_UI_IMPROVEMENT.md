# 商城商品卡片 UI 改进

## 📋 改进说明

将商品卡片上的三个操作按钮从"悬停时出现"改为"始终显示"，提升用户体验和操作便利性。

## 🔄 改进对比

### 改进前
- ❌ 三个圆形按钮悬停时从底部滑入
- ❌ 不悬停时看不到操作按钮
- ❌ 移动端体验不佳（没有悬停状态）
- ❌ 用户不知道有这些功能

### 改进后
- ✅ 操作按钮始终显示在商品信息底部
- ✅ 主按钮：加入购物车（带文字+图标）
- ✅ 辅助按钮：查看详情、收藏（圆形图标）
- ✅ 清晰的视觉层次
- ✅ 移动端友好

## 🎨 新设计

### 布局结构
```
┌─────────────────────┐
│   商品图片区域       │
│   (点击查看详情)     │
├─────────────────────┤
│ 分类标签            │
│ 商品名称            │
│ ¥价格    已售xxx    │
├─────────────────────┤
│ [加入购物车] 👁 ⭐  │
└─────────────────────┘
```

### 按钮说明

1. **加入购物车** - 主操作按钮
   - 样式：红色主题按钮
   - 位置：占据大部分宽度
   - 图标：🛒 ShoppingCart
   - 文字：加入购物车

2. **查看详情** - 辅助操作
   - 样式：圆形按钮
   - 图标：👁 View
   - 悬停提示：查看详情

3. **收藏** - 辅助操作
   - 样式：圆形按钮
   - 图标：⭐ Star
   - 悬停提示：收藏

## 🎯 用户体验改进

### 1. 可发现性提升
- **改进前**：用户需要悬停才能发现操作按钮
- **改进后**：操作按钮始终可见，用户一眼就能看到

### 2. 操作便利性
- **改进前**：需要精确悬停在商品卡片上
- **改进后**：直接点击按钮即可，无需悬停

### 3. 移动端适配
- **改进前**：移动端没有悬停状态，按钮无法显示
- **改进后**：移动端也能正常显示和使用按钮

### 4. 视觉层次
- **改进前**：三个按钮平级，不分主次
- **改进后**：主按钮突出，辅助按钮收敛

## 📱 响应式设计

### 桌面端 (>992px)
- 加入购物车按钮：占据大部分宽度
- 辅助按钮：两个圆形图标并排

### 平板端 (768px-992px)
- 保持相同布局
- 按钮尺寸略小

### 移动端 (<768px)
- 加入购物车按钮：全宽显示
- 辅助按钮：可能隐藏或移到其他位置
