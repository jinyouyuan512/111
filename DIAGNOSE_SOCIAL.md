# 社区功能诊断指南

## 第一步：确认访问地址

**重要！** 前端端口已变更为 3002

请在浏览器地址栏确认你访问的是：
```
http://localhost:3002
```

如果是 3001，那就是旧版本，功能当然没有！

## 第二步：强制刷新

按以下步骤操作：

### Windows/Linux:
1. 按 `Ctrl + Shift + Delete`
2. 选择"缓存的图片和文件"
3. 点击"清除数据"
4. 然后按 `Ctrl + Shift + R` 强制刷新

### Mac:
1. 按 `Cmd + Shift + Delete`
2. 选择"缓存的图片和文件"
3. 点击"清除数据"
4. 然后按 `Cmd + Shift + R` 强制刷新

## 第三步：检查代码是否加载

1. 按 `F12` 打开开发者工具
2. 切换到 "Sources" 或 "源代码" 标签
3. 找到 `src/views/Social.vue`
4. 搜索 "StarFilled"
5. 如果找不到，说明代码没有加载

## 第四步：检查具体问题

### 检查点赞按钮

在浏览器中：
1. 右键点击点赞按钮
2. 选择"检查元素"
3. 查看HTML代码

**应该看到**:
```html
<div class="interaction-btn">
  <i class="el-icon">
    <svg>...</svg>  <!-- 这里应该有SVG图标 -->
  </i>
  <span>342</span>
</div>
```

**如果看到**:
```html
<div class="interaction-btn">
  <span>342</span>  <!-- 只有数字，没有图标 -->
</div>
```
说明代码没有加载！

### 检查评论功能

在浏览器控制台（F12 → Console）输入：
```javascript
console.log(typeof showComments)
```

**应该输出**: `function`
**如果输出**: `undefined` 说明代码没有加载！

### 检查发布框按钮

1. 点击"发布动态"
2. 右键点击图片按钮
3. 选择"检查元素"
4. 查看是否有 `disabled` 属性

**应该看到**:
```html
<button class="el-button">
  <!-- 没有 disabled 属性 -->
</button>
```

**如果看到**:
```html
<button class="el-button" disabled>
  <!-- 有 disabled 属性 -->
</button>
```
说明代码没有加载！

## 第五步：重启前端

如果以上都不行，重启前端：

```bash
# 停止前端（在运行前端的终端按 Ctrl+C）

# 重新启动
cd frontend
npm run dev
```

等待启动完成，查看输出的端口号，访问正确的地址。

## 第六步：验证功能

### 验证点赞
1. 找到任意动态
2. 查看点赞按钮是否有星星图标
3. 点击后图标是否变色

### 验证评论
1. 点击评论图标
2. 是否打开弹窗
3. 能否输入和发表评论

### 验证发布框
1. 点击"发布动态"
2. 点击图片按钮
3. 点击视频按钮
4. 两个按钮是否都可以点击（不是灰色）

## 常见问题

### Q: 我访问的就是 3002，但还是没变化
A: 清除浏览器缓存后强制刷新（Ctrl + Shift + R）

### Q: 清除缓存后还是没变化
A: 重启前端服务

### Q: 重启后还是没变化
A: 检查前端是否有编译错误，查看终端输出

### Q: 终端显示编译成功，但功能还是没有
A: 可能是浏览器扩展干扰，尝试无痕模式
